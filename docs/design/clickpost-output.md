# 配送管理画面への出力 / 注入設計

対象: クリックポストなど、ブラウザ上で配送先を手入力する外部の配送管理画面へ、
LogisticUtils で抽出した配送先データをどう渡すか。

## 検討した方式

### 1. クリップボード経由のコピー(TSV/CSV)
- 抽出結果をタブ区切り(TSV)でクリップボードにコピーし、対象画面の入力欄に手動で貼り付けてもらう。
- 実装コストが最も低く、対象サイトの DOM 構造に一切依存しないため、
  クリックポスト側の画面変更にも影響を受けない。
- 一方でフィールドへの貼り付け自体は手動操作が残る。

### 2. アプリ専用の Chrome を CDP (Chrome DevTools Protocol) で制御し、フォームへ注入
- Rust 側で [`chromiumoxide`](https://github.com/mattsse/chromiumoxide)(Puppeteer 相当の CDP クライアント)を用い、
  アプリ専用のプロファイルで Chrome/Chromium を起動する。
- 認証はユーザーがその起動されたウィンドウ内で手動で行う(アプリはログイン情報を扱わない)。
  プロファイルはアプリのデータフォルダ配下に永続化するため、次回以降の再ログインは基本的に不要。
- 対象フォームへの反映は、ユーザーが編集する JSON 設定ファイル
  (`clickpost-field-map.json`、アプリの設定フォルダ配下)で「抽出項目名 → CSS セレクタ」を
  マッピングし、ボタン押下時にそのセレクタへ `value` を設定 + `input`/`change` イベントを発火する。
- クリックポストの実際の入力欄のセレクタは未調査のため、初回はユーザーが対象画面を
  ブラウザの開発者ツールで調べてセレクタを設定する必要がある。サイト側の変更で
  セレクタが壊れた場合も、設定ファイルの更新のみで追従できる。

### 3. ブラウザ拡張との連携
- 別途ブラウザ拡張を開発し、LogisticUtils とはクリップボードやローカルファイル経由で連携する。
- スコープが LogisticUtils(Tauri デスクトップアプリ)を超えるため、優先度は低い。

## 採用方針

- **フェーズ1(実装済み)**: クリップボードへの TSV コピー機能を UI に追加する。
  値の列のみをタブ区切りでコピーすることで、複数フィールドへの一括貼り付けに対応できる
  Web フォーム(表形式ペースト対応のもの)であれば手作業なしで反映できる。
- **フェーズ2(実装済み・要セレクタ調査)**: 方式2(CDP 経由の自動入力)を実装。
  - Rust: `src-tauri/src/browser.rs` にブラウザセッション管理と注入ロジックを実装。
    Tauri コマンド `open_clickpost_browser` / `inject_rows` として公開。
  - フロント: `App.svelte` に「ClickPostを開く」「データを連携」ボタンを追加(Tauri 環境でのみ表示)。
  - 認証は起動された Chrome ウィンドウ内でユーザーが手動実施する運用とし、
    アプリはログイン情報を一切扱わない。
  - 実際の入力欄のセレクタは未調査のため、初期状態の `clickpost-field-map.json` は
    プレースホルダのセレクタが入っている。実運用前にユーザーが実画面を調査して更新する必要がある。
